Проект по SQL. База данных онлайн книг¶

Описание проекта: Коронавирус застал мир врасплох, изменив привычный порядок вещей. На какое-то время жители городов перестали выходить на улицу, посещать кафе и торговые центры. Зато стало больше времени для книг. Это заметили стартаперы — и бросились создавать приложения для тех, кто любит читать. Ваша компания решила быть на волне и купила крупный сервис для чтения книг по подписке. Ваша первая задача как аналитика — проанализировать базу данных.

Цель исследования: оценка ценностного предложения для сервиса для чтения книг по подписке.

Исследовательские задачи¶

  • Посчитать, сколько книг вышло после 1 января 2000 года;
  • Для каждой книги посчитать количество обзоров и среднюю оценку;
  • Определить издательство, которое выпустило наибольшее число книг толще 50 страниц — так мы исключим из анализа брошюры;
  • Определить автора с самой высокой средней оценкой книг — учитывать только книги с 50 и более оценками;
  • Посчитать среднее количество обзоров от пользователей, которые поставили больше 48 оценок.

Описание данных¶

Таблица books cодержит данные о книгах:

  • book_id — идентификатор книги;
  • author_id — идентификатор автора;
  • title — название книги;
  • num_pages — количество страниц;
  • publication_date — дата публикации книги;
  • publisher_id — идентификатор издателя.

Таблица authors cодержит данные об авторах:

  • author_id — идентификатор автора;
  • author — имя автора.

Таблица publishers cодержит данные об издательствах:

  • publisher_id — идентификатор издательства;
  • publisher — название издательства;

Таблица ratings cодержит данные о пользовательских оценках книг:

  • rating_id — идентификатор оценки;
  • book_id — идентификатор книги;
  • username — имя пользователя, оставившего оценку;
  • rating — оценка книги.

Таблица reviews cодержит данные о пользовательских обзорах:

  • review_id — идентификатор обзора;
  • book_id — идентификатор книги;
  • username — имя автора обзора;
  • text — текст обзора.

books_rating = query('''
WITH
book_ratings AS
(SELECT  book_id,
        ROUND(AVG(rating), 1) AS avg_rating
FROM ratings
GROUP BY book_id),

book_review_cnt AS
(SELECT  book_id,
        COUNT(review_id) AS review_cnt
FROM reviews
GROUP BY book_id)

SELECT  b.book_id,
        title,
        avg_rating,
        COALESCE(review_cnt, 0) AS review_cnt
FROM books AS b
LEFT OUTER JOIN book_ratings AS br ON b.book_id=br.book_id
LEFT OUTER JOIN book_review_cnt AS brc ON b.book_id=brc.book_id
ORDER BY avg_rating DESC
''')

query('''
SELECT  publisher,
        COUNT(b.book_id) AS total_cnt

LEFT OUTER JOIN publishers AS p ON b.publisher_id=p.publisher_id
WHERE num_pages > 50
GROUP BY publisher
ORDER BY total_cnt DESC
LIMIT 5

Определим автора с самой высокой средней оценкой книг — учтем только книги с 50 и более оценками.

In [11]:
query('''
WITH
avg_book_ratings AS 
(SELECT author_id,
        b.book_id,
        AVG(rating) AS avg_rating,
        COUNT(rating_id) AS rating_cnt
FROM ratings AS r
LEFT OUTER JOIN books AS b ON b.book_id=r.book_id
GROUP BY author_id, b.book_id
HAVING COUNT(rating_id) >= 50
ORDER BY avg_rating DESC),

avg_author_rating AS
(SELECT  author_id,
        AVG(avg_rating) AS avg_author_rating
FROM avg_book_ratings
GROUP BY author_id
ORDER BY avg_author_rating DESC)

SELECT  author,
        ROUND(avg_author_rating, 2) AS avg_rating
FROM authors as a
RIGHT OUTER JOIN avg_author_rating AS aar ON aar.author_id=a.author_id
ORDER BY avg_author_rating DESC
LIMIT 5
''')

                    GROUP BY username
